{% comment %} Implemented features of the TOC include: * It is usable as a **Liquid include**, for example: ```liquid {% include toc_chart.html content=content %} ``` and it also supports an optional mode flag: ```liquid {% include toc_chart.html content=content compact=true %} ``` * It reads the rendered HTML content and builds a TOC from **`h1`**, **`h2`**, and **`h3`** headings. * **`h1`** headings are rendered on their own line and centered. * **`h2`** groups are visually separated with a light box so each section is easier to distinguish. * When an **`h2`** has child **`h3`** headings, the TOC uses a two-column layout: * left column: the **`h2`** * right column: all **`h3`** entries that belong to that **`h2`** * The left **`h2`** column is narrower than before, and the **`h2`** text is vertically centered. * When an **`h2`** has **no `h3`** under it, it is shown alone as a single-row item instead of forcing a two-column layout. * A **compact mode** is supported for grouped **`h3`** items: * multiple **`h3`** entries are displayed inline after the **`h2`** * each **`h3`** stays unbroken as one unit * wrapping can occur only between items, not inside one item * Multiple **`h1`** headings are now handled in document order, so each **`h1`** is followed by its own related content instead of all **`h1`** items being collected at the top. * Compatibility logic has been added for irregular heading structures, including: * **`h1`** followed directly by **`h3`** * documents containing only **`h3`** * mixed structures such as **`h1 > h2 > h3`** followed later by **`h1 > h3`** * Standalone **`h3`** headings, when there is no active parent **`h2`**, are rendered as their own single-row entries. * The implementation was adjusted to avoid the earlier parsing bug caused by separators such as horizontal rules rendered as **`
`**. * Styling is designed to **inherit** the site’s existing typography and general appearance as much as possible, instead of imposing a heavy custom theme. * Links are generated from heading **`id`** attributes when available, so headings can be clicked in the TOC.
