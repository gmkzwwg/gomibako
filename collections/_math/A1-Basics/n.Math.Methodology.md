## Part 1. 数学能力

### 数学能力是什么？

数学能力不是公式数量、做题速度、阅读页数，也不是能否复述一段标准证明。一个人知道`勾股定理`，只能说明他掌握了一个数学事实；能够判断什么时候把问题转成直角三角形，什么时候引入坐标，什么时候改用向量、内积或投影，才说明这个事实已经进入能力系统。

更准确地说，数学能力由`数学事实`、`数学工具`、`数学世界模型`和`数学直觉`共同构成。`数学事实`提供材料，`数学工具`提供操作，`数学世界模型`提供方向感，`数学直觉`是在具体问题中快速调动这些东西的能力。

`数学事实`包括概念、定义、定理、公式、标准结果、典型例子和反例。学习分析学时，`极限`、`连续`、`一致收敛`、`紧致性`是事实；学习线性代数时，`特征值`、`特征多项式`、`相似`、`秩`、`核`、`像`是事实；学习代数学时，`群`、`环`、`域`、`同态`、`商结构`是事实。没有事实，思考没有对象。

`数学工具`是可执行的操作。它不是某个公式本身，而是围绕某类问题形成的方法。比如把含参数不等式转化为函数最值问题；固定一个参数，把多元问题降成一元问题；先证明可逆情形，再用逼近和连续性过渡到一般情形；先做特殊情形，再回到一般情形；用反例测试命题边界。工具不是“知道有这个方法”，而是知道它何时出现、怎样执行、能产出什么、在哪里失效。

`数学世界模型`是压缩和预测。学习者长期与数学对象、例子、证明、失败经验互动后，会逐渐形成判断：这个量增长快还是慢，哪个条件可能关键，哪条路线更轻，某种语言是否更自然，一个方法最后大概能给出强结论还是弱估计。世界模型不等于证明，但它决定第一步往哪里走。

`数学直觉`是世界模型在具体问题中的输出。直觉不是神秘灵感，而是大量事实、工具、例子、反例和试错经验压缩后的快速判断。看到一个不等式，先想到单调性、凸性、放缩、取对数还是积分估计；看到一个矩阵问题，先想到相似、秩、特征多项式、可逆近似还是标准形；看到一个极限问题，先想到主项、误差、换元还是一致性条件。这些判断看起来像“感觉”，实际背后是训练留下的结构记忆。

**数学思维不是魔法，而是日常问题解决能力的延伸。数学家处理问题时，会抽象、分解、类比、寻找连接，也会尝试不完全理解的路径，并从失败中获得信息。**

### 为什么事实不等于能力？

事实只回答“有什么”，能力还要求回答“怎么用”。只知道`紧致性`的定义，只能说明知道“开覆盖存在有限子覆盖”这个表述；真正理解紧致性，需要知道它经常承担什么功能：把无限信息压缩成有限信息，把局部控制转成全局控制，把序列逃逸问题转成收敛子列问题。

同样，只知道`特征多项式`的定义，不等于会处理矩阵问题。若要证明 `PQ` 和 `QP` 有相同的特征多项式，直接展开矩阵元素通常很重。更有结构的路线是先处理可逆情形：可逆性带来相似关系，相似关系保留特征多项式；一般情形再通过“可逆矩阵在全体矩阵中稠密”以及“行列式作为矩阵元素的多项式是连续的”来传递结论。这里真正有价值的不是记住这一题，而是获得一个方法原型：先在好对象上证明，再把结论传回一般对象。

事实进入能力系统，通常会发生这样的变化：

* `紧致性` → 识别“无限转有限”的机会。
* `一致收敛` → 判断极限、积分、连续性是否可以交换或保留。
* `特征多项式` → 把矩阵问题转成不变量问题。
* `凸性` → 控制割线、切线、极值和 Jensen 型不等式。
* `同态` → 把复杂结构映射到更容易处理的结构。
* `反例` → 标出定理条件的边界。

箭头左侧是事实，右侧是能力入口。只记左侧，学习停留在材料层；能说出右侧，事实开始转化为工具。

### 工具怎样进入能力？

工具进入能力，不靠“听说过”，而靠反复使用。第一次看到一个方法，只是知道它存在；第二次在提示下用出来，说明它开始有印象；第三次在新题中主动识别触发信号，才说明它开始成为自己的工具。

一个工具至少要保存四类信息：

* `适用对象`：它处理什么问题。
* `触发信号`：题面出现什么结构时应想到它。
* `执行动作`：它把问题改写成什么形态。
* `失败边界`：它在什么情况下不能用，或只能给出弱结果。

例如`比值判别法`不能只记成一个公式。更有效的记录方式是：它适合处理正项级数或绝对收敛问题，尤其适合相邻项比值容易化简的级数；阶乘、指数、幂次混合时，常常值得先试。它的边界也必须一起记住：当极限等于 $$1$$ 时，它无法判断，此时可能需要`Raabe 判别法`、比较判别法、积分判别法或渐近展开。

再看一个含参数不等式。题面可能只是要求证明一个不等式，但工具层的处理会更清楚：先把不等式改写成函数问题，再判断是否需要求最值；如果函数有两个变量，直接处理多元最值可能很重，可以固定一个参数，把曲面切成一元曲线，再看单调性。图像先帮助猜方向，严格证明再处理导数、放缩和边界。

工具的价值在于增加操作空间。没有工具时，问题只有“会”和“不会”；有工具后，问题会出现多个入口：换语言、看边界、做特殊情形、构造反例、寻找不变量、先处理好对象、再过渡到一般对象。

### 数学世界模型有什么用？

数学世界模型让学习者在没有完整证明之前，先获得方向感。它不能替代证明，但能减少盲目尝试。

在分析学中，看到`紧致区间`，应想到有限覆盖、最大最小值、收敛子列、一致连续；看到`非紧空间`，应警惕最大值不存在、序列逃逸、局部控制无法变全局控制；看到`逐点收敛`，应警惕它不一定保留连续性、积分交换和求导交换；看到`一致收敛`，才可以期待更多极限过程被保留。

在线性代数中，看到`相似矩阵`，应想到特征多项式、迹、行列式、特征值保持；看到矩阵乘法，应警惕一般情形下不可交换；看到`可逆矩阵`，应想到很多证明会更容易，也应想到一般矩阵有时可以由可逆矩阵逼近；看到`秩`，应联想到核、像、线性方程组可解性和维数关系。

世界模型越好，越能判断“哪条路可能有戏”。这种判断不等于正确答案，但它能决定第一步往哪里走。没有世界模型时，学习者容易在每道题前重新开始；有世界模型后，新题会自动关联到旧对象、旧方法和旧失败。

### 直觉从哪里来？

数学直觉不是和严格证明相反的东西。直觉负责提出方向，证明负责检查方向。没有直觉，问题搜索成本很高；没有证明，直觉容易失控。

直觉来自可压缩经验。一个学习者见过足够多的函数图像，就会对单调性、凸性、极值和渐近行为有预期；做过足够多的级数，就会对指数、阶乘、幂次、对数的增长强弱有感觉；处理过足够多的反例，就会知道哪些条件不能随便删。直觉不是“突然知道答案”，而是“某种结构看起来像以前见过的结构”。

这也解释了为什么例子库和反例库重要。只有抽象定义，没有例子，概念是空的；只有正例，没有反例，概念边界是模糊的。学习`连续`，需要标准连续函数，也需要不连续函数；学习`一致连续`，需要知道它和连续的区别；学习`可微`，需要知道可微推出连续，也需要知道连续不推出可微；学习`紧致`，需要知道闭区间紧致，也需要知道开区间、无界闭集为什么不紧致。

直觉还需要数学实验支持。图像、数值、特殊值、随机样本、符号计算，都可以帮助直觉快速获得反馈。计算不能替代证明，但能暴露错误方向。例如一个不等式猜想若在随机参数下反复失败，应先找反例，而不是继续包装证明；一个极限猜想若数值震荡严重，应检查是否缺少一致性、单调性或支配条件。

### 数学家和学习者的差别在哪里？

数学家并不是不失败，也不是总能直接看见正确路线。更真实的差别在于：数学家更擅长把失败转化为信息。

学习者常把失败概括成“不会”。这个说法没有训练价值。更有效的失败描述应当细到可以行动：

* 定义无法展开。
* 不知道该选哪个定理。
* 只能证明特殊情形。
* 估计太粗，误差压不下去。
* 构造对象时无法保证条件。
* 证明中默认了可逆性、连续性或可交换性。
* 直觉认为命题为真，但找不到支持例子。
* 答案用了某个工具，但自己没有识别出触发信号。

数学家的优势往往在这些细节上。他们能更早意识到一条路线哪里重，哪里可能失败，哪里需要换语言，哪里应该找反例，哪里值得暂时强化条件。他们不会把每次失败都当成自我评价，而会把失败拆成关于问题的信息。

这也是数学训练中最应保存的部分。答案只告诉人一条成功路线，失败记录能告诉人搜索空间的形状。

### 怎样判断数学能力提升了？

数学能力的提升不一定表现为做题速度立刻变快。更可靠的标志是判断质量提高。

* “不会” → 卡点是什么。
* 只看答案 → 还原答案为什么这样进入。
* 背定理 → 定理的触发信号。
* 做一道题 → 抽出一类题的结构。
* 接受证明 → 检查证明是否合法。
* 相信直觉 → 用例子、反例、计算或形式条件验证直觉。
* 看见复杂问题只能停住 → 先处理特殊情形、边界情形或简化模型。

这些变化比“今天多背了几个公式”更重要。速度可能后来才出现，甚至在某些阶段会暂时变慢，因为学习者开始检查条件、比较路线、保存反例、复盘失败。短期看，这会增加步骤；长期看，它会提高判断质量。

真正稳定的数学能力，不是把每道题都快速做完，而是面对新问题时能形成一组有秩序的动作：先读对象，再读条件，再改写目标；先看特殊情形，再找工具；先生成猜想，再用例子、反例、计算和证明检查；最后把经验压缩成下次可以调用的东西。

## Part 2. 做数学

### 为什么只读书不够？

数学教材可以提供高质量材料，但阅读本身不是完整训练。读者在阅读中获得的是定义、定理、证明和例子；数学能力还要求与这些对象发生互动。没有互动，理解很容易停在“看懂了”的层面。

“看懂”有很多层。能跟着证明走一遍，是最低层的看懂；能解释每一步为什么需要，是更高一层；能删掉一个条件并构造反例，是再高一层；能在另一个问题中重新使用这段证明结构，才接近真正掌握。

只读书容易产生顺滑错觉。教材中的证明通常已经被清理过：错误路线被删掉，动机被压缩，关键变形直接出现，细节被安排在合适位置。读者顺着它走，会觉得每一步都合理；一旦独立面对相似问题，却不知道第一步该做什么。

**数学学习不能只靠输入。定义、定理和证明必须经过尝试、失败、修改、验证和复盘，才会从“读过的内容”变成“可调用的能力”。**

例如学习`一致收敛`时，只读定义和定理，很容易以为自己已经理解。真正的训练发生在具体问题中：给出一列函数，判断是逐点收敛还是一致收敛；尝试交换极限和积分；寻找逐点收敛但不保连续性的例子；检查定理中每个条件为什么存在。到这个阶段，`一致收敛`才不再只是定义，而成为处理极限过程的工具。

### 什么叫做数学？

做数学不等于机械刷题，也不等于重复计算。更准确地说，做数学是一种主动交互：给定对象，尝试操作；观察反馈，修正理解；遇到失败，定位原因；得到结果，压缩经验。

一次真正的数学活动至少包含四个动作：

* 接受或提出一个明确问题。
* 把问题转写成可操作形式。
* 调用事实、工具、例子和反例进行尝试。
* 检查结果，并把经验沉淀下来。

做题是最常见的入口，但不是唯一入口。读证明时补跳步，给定定义时构造例子，看到命题时寻找反例，用计算工具测试猜想，把自然语言证明改写成形式化证明，这些也都是做数学。

比如读到“连续函数在紧致集上取到最大值和最小值”时，真正的训练不止是记住定理。可以继续做几件事：

* 把紧致集换成开区间，看结论是否失败。
* 把连续函数换成不连续函数，看结论是否失败。
* 在 $$[0,1]$$ 上构造简单例子，观察最大值在哪里出现。
* 在 $$(0,1)$$ 上考察 $$f(x)=x$$ 或 $$f(x)=1/x$$，理解为什么边界会造成问题。
* 回到证明，检查`紧致性`和`连续性`分别在哪一步被使用。

这才是一个定理进入能力系统的过程。

### 为什么问题是训练单位？

问题能暴露真实缺口。只读定义时，很难知道自己是否理解`一致连续`；面对具体证明时，才会发现自己不清楚它与`连续`的差别。只背`紧致性`定义时，很难知道它有什么用；处理一个需要从无穷局部信息中抽出有限控制的问题时，紧致性才从事实变成工具。

问题还会迫使工具竞争。同一道题可能有多条路线：代数化、几何化、估计化、概率化、构造反例、引入不变量。工具竞争会训练选择能力。没有问题，工具只是清单；进入问题，工具才有优劣、成本、边界和适用条件。

例如证明一个含参数不等式时，至少有几种可能入口：

* 直接求导。
* 固定参数，降成一元函数。
* 取对数，降低指数复杂度。
* 看边界值和等号情形。
* 用凸性或 Jensen 型不等式。
* 用图像或数值实验先判断方向。

哪条路更自然，不能只靠背书决定。它要通过问题本身的结构、变量范围、目标形式和失败反馈来判断。

问题还有一个重要作用：生成记忆钩子。抽象定理很难孤立记住，但一旦绑定到几个典型问题，就会变成可调用经验。一个好问题会同时留下事实、工具、直觉和反例。

### 一次训练怎样开始？

训练开始前，不急着找答案。先把问题拆成可观察对象。

| 步骤   | 要做什么                                    | 目的        |
| ---- | --------------------------------------- | --------- |
| 读对象  | 判断对象是数列、函数、集合、矩阵、群、图、随机变量，还是空间          | 决定可用语言    |
| 读条件  | 标出`连续`、`可微`、`紧致`、`有界`、`单调`、`可逆`、`交换`等条件 | 找工具触发信号   |
| 读目标  | 判断要证明等式、不等式、存在性、唯一性、收敛、连续、可积，还是构造反例     | 决定证明形态    |
| 做小版本 | 先看低维、特殊参数、边界点、有限情形、对角矩阵、简单函数            | 暴露结构      |
| 记卡点  | 写清第一处卡住的位置                              | 为后续反馈提供材料 |

这五步能把“我不会”拆成更具体的问题。训练的第一目标不是立刻做出题，而是让问题变得可诊断。

例如遇到一个矩阵命题，不要马上展开元素计算。可以先问：

* 是否只与`迹`、`行列式`、`秩`、`特征多项式`有关？
* 是否在相似变换下保持不变？
* 是否可以先看对角矩阵？
* 是否可以先看可逆矩阵？
* 是否可以先看 $$2 \times 2$$ 情形？
* 是否存在非交换风险？

这些问题能快速决定方向。即使最后没有做出证明，也会留下比“不会”更有价值的信息。

### 什么时候该看答案？

过早看答案会切断训练。无限硬扛也会浪费时间。更合适的做法是限时独立尝试，然后带着明确卡点去看提示或答案。

不同任务可以设置不同时间：

| 任务类型        |   独立尝试时间 | 目标             |
| ----------- | -------: | -------------- |
| 基础定义题       |  5—10 分钟 | 检查定义是否能展开      |
| 常规计算题       | 10—20 分钟 | 检查方法是否熟练       |
| 中等证明题       | 20—40 分钟 | 找入口和关键条件       |
| 综合题         |   1—2 小时 | 形成路线图和失败记录     |
| 高难题 / 研究型问题 |   多次分段尝试 | 建立问题档案，而不是一次解决 |

看答案前，至少应留下自己的尝试痕迹：

* 题目目标。
* 已知条件。
* 试过的工具。
* 失败位置。
* 一个特殊情形。
* 一个反例方向。
* 一个仍然不确定的问题。

没有这些记录，答案会变成顺滑叙述；有了这些记录，答案才会变成诊断材料。

### 答案应该怎样使用？

答案不是抄写对象，而是拆解对象。看答案时，最重要的不是“它怎么写完”，而是“它为什么这样开始”。

可以按这组问题拆答案：

* 第一行为什么这样写？
* 它把目标改写成了什么形式？
* 它使用了哪个条件？
* 它绕开了什么困难？
* 它有没有先处理特殊情形？
* 它用了哪个标准工具？
* 哪一步最不可替代？
* 哪一步是技术计算，哪一步是结构想法？
* 如果删掉某个条件，这个证明会在哪里断？

例如一道证明题的答案第一步是“令 $$g(x)=f(x)-x$$”。不要只记这个操作。应问：为什么要构造这个函数？原题是不是有“不动点”“交点”“等式 $$f(x)=x$$”这类目标？构造后是否把目标变成了 $$g(x)=0$$？如果是，那么这个动作就可以压缩成工具：遇到不动点问题时，常可把它改写成零点问题，再考虑介值定理、单调性或压缩映射。

答案中的关键动作要转化为工具卡，而不是只写进错题本。错题本记录“这题错了”；工具卡记录“以后遇到什么信号可以这样做”。

### 怎样把一道题变成一类题？

做出一道题不是终点。更重要的是从题目中提取可复用结构。

例如一道含参数不等式，若只是保存完整证明，收益有限。更有价值的提取方式是：

`含参数不等式` → `函数化` → `固定参数` → `一元单调性` → `边界控制`

这条链可以迁移到很多题。以后遇到类似命题，学习者可以先尝试同样的结构，而不是重新从零开始。

再如矩阵问题中，“先证明可逆情形，再过渡到一般情形”也应被保存为一类结构：

`一般对象难处理` → `好对象上结论清楚` → `好对象在大空间中稠密` → `相关运算连续` → `结论传回一般对象`

这个结构不仅能用于矩阵，还能在分析、拓扑、泛函分析中反复出现。学习的重点不是记住某道题，而是识别这些跨问题结构。

一道题完成后，可以用下面的方式压缩：

| 提取项   | 要回答的问题          |
| ----- | --------------- |
| `题型`  | 它属于哪类问题？        |
| `对象`  | 处理的数学对象是什么？     |
| `目标`  | 要证明什么形态的结论？     |
| `入口`  | 第一有效动作是什么？      |
| `工具`  | 用了哪些方法？         |
| `条件`  | 哪些条件真正被使用？      |
| `失败路` | 哪些路线看似自然但走不通？   |
| `反例`  | 哪些条件删掉会失败？      |
| `迁移`  | 哪些相似问题可以使用同一结构？ |

这个表比单纯抄答案更有用。它把题目变成训练材料。

### 为什么训练需要复现？

第一次做出一道题，常常只是短时成功。几天后重新做，如果仍然找不到入口，说明方法还没有进入系统。复现训练的目标不是背答案，而是复现入口和关键转折。

复现可以分三类：

* 隔天复现：检查是否记得主要路线。
* 隔周复现：检查是否能从题面重新触发工具。
* 变式复现：检查是否能迁移到相邻问题。

真正有价值的是变式复现。原题会做，可能只是记忆；变式能做，才说明工具开始稳定。

例如原题证明某个具体函数列一致收敛，变式可以换成另一组函数；原题处理 $$[0,1]$$，变式可以改成 $$(0,1)$$；原题给出连续函数，变式可以去掉连续性，看结论是否还成立。变式会暴露方法到底被理解，还是只被记住。

### 数学训练怎样留下产物？

一次训练结束时，至少应留下一个可复用对象。没有产物，训练很容易只停留在短时理解。

| 新增内容 | 应留下的产物      | 例子                   |
| ---- | ----------- | -------------------- |
| 新概念  | 定义卡 / 概念辨析卡 | `一致收敛`与`逐点收敛`的区别     |
| 新定理  | 定理卡 / 条件卡   | `介值定理`需要连续性          |
| 新方法  | 工具卡         | 不动点问题 → 零点问题         |
| 新错误  | 错因卡         | 默认了极限与积分可交换          |
| 新边界  | 反例卡         | 逐点收敛不保连续             |
| 新结构  | 题型模板        | 含参数不等式 → 函数最值        |
| 新判断  | 直觉笔记        | 非紧空间中警惕最大值不存在        |
| 新连接  | 笔记链接        | `紧致性` ↔ 有限子覆盖 ↔ 收敛子列 |

产物不需要很长，但要能在未来被调用。一个有效产物通常满足三个条件：短、明确、可测试。

例如“今天学了紧致性”不是好产物；“紧致性常用于把无穷局部信息压缩成有限全局控制”更好；“若证明需要从每个点附近的局部估计推出全局统一估计，先检查是否有紧致性”则已经接近工具卡。

### Part 2 的工具卡：一次数学训练记录表

| 模块    | 记录内容         | 示例                      |       |    |
| ----- | ------------ | ----------------------- | ----- | -- |
| 题目    | 原题或简写        | 证明某函数列在 $$[0,1]$$ 上一致收敛 |       |    |
| 对象    | 数列、函数、矩阵、集合等 | 函数列                     |       |    |
| 条件    | 题目给出的关键条件    | 区间闭且有界；函数连续             |       |    |
| 目标    | 要证明的结论形式     | 一致收敛                    |       |    |
| 第一入口  | 最先尝试的动作      | 写出上确界范数                 |       |    |
| 尝试路线  | 试过哪些方法       | 逐点极限、估计上界、看端点           |       |    |
| 卡点    | 具体卡在哪里       | 无法控制 $$\sup             | f_n-f | $$ |
| 特殊情形  | 做过哪些小版本      | 先看 $$x=0$$、$$x=1$$、中点   |       |    |

这张表的作用，是把一次做题变成一次训练。题目不再只是“做完 / 没做完”，而是被拆成对象、条件、目标、工具、卡点、反例和迁移结构。长期看，这些记录会形成自己的数学工具库。
## Part 3. 选题

### 为什么选题本身就是学习能力？

数学训练不是随机消耗题目。题目选得太简单，只会重复已有动作；题目选得太难，容易只剩下看答案；题目选得太散，经验无法沉淀成方法。有效训练需要题目处在一个合适位置：它比当前能力略高，但仍然能通过尝试、提示、查证和复盘进入理解范围。

选题的目标不是“越难越好”，而是让题目承担明确功能。有些题用于巩固定义，有些题用于训练工具，有些题用于暴露反例，有些题用于连接章节，有些题用于练习证明写作。题目本身没有绝对好坏，关键看它是否服务于当前训练目标。

初学阶段最常见的问题，是过早追求综合题。概念还没有例子，定理条件还不熟，基本计算还不稳定，却直接做高难题，结果往往变成“看答案学习”。答案当然能提供信息，但缺少独立尝试时，学习者很难知道自己的真实缺口在哪里。

更合理的顺序是：

`定义题` → `例子题` → `反例题` → `工具题` → `综合题` → `开放题`

这个顺序不是固定路线，但它提醒一件事：数学能力需要层层安装。没有定义翻译和例子基础，综合题中的方法很难真正进入系统。

### 题目有哪些类型？

数学题可以按功能分类。分类的目的不是给题目贴标签，而是知道做这道题时要训练什么。

| 类型    | 训练目标        | 常见形式                  | 做题重点        |
| ----- | ----------- | --------------------- | ----------- |
| `定义题` | 把概念展开成可操作语言 | 判断某对象是否满足定义           | 不跳过量词、条件、边界 |
| `例子题` | 给概念配具体对象    | 构造满足条件的对象             | 找最简单、最典型的模型 |
| `反例题` | 理解定理边界      | 否定一个错误命题              | 找边界、退化、极端情形 |
| `计算题` | 稳定基本技术      | 求极限、积分、矩阵、特征值         | 准确、熟练、能查错   |
| `工具题` | 训练某个方法      | 用某定理或技巧证明结论           | 识别触发信号      |
| `连接题` | 打通章节关系      | 用线性代数处理微分方程，用拓扑处理分析问题 | 换语言、找结构     |
| `综合题` | 同时调用多个工具    | 长证明、多步骤问题             | 组织路线，控制复杂度  |
| `开放题` | 训练猜想、探索、研究感 | 推广、分类、寻找更强结论          | 生成问题，测试边界   |

学习者常把所有题都叫“刷题”，这会遮蔽训练差异。做`定义题`时，重点不是速度，而是量词和条件；做`反例题`时，重点不是套方法，而是理解概念边界；做`综合题`时，重点不是每一步都新奇，而是能把多个旧工具组织起来。

### 初学者应该选什么题？

初学者最需要的是定义、例子和基本工具，而不是高难综合题。一个概念如果没有例子，就没有附着点；如果没有反例，就没有边界；如果没有基本题，就无法进入后续证明。

学习`连续`时，不应只背定义。更好的选题顺序是：

* 判断多项式、三角函数、绝对值函数是否连续。
* 判断分段函数在拼接点是否连续。
* 构造一个在某点不连续的函数。
* 构造一个处处连续但不可导点很多的函数。
* 比较`连续`、`一致连续`、`可微`之间的关系。
* 解释为什么连续函数在闭区间上有最大最小值。

这个顺序先让概念落地，再逐渐进入工具。若一开始直接证明抽象定理，学习者可能能跟着证明走，却不知道证明到底在保护什么。

学习线性代数也类似。刚学`特征值`时，不宜立刻跳到 Jordan 标准形的复杂问题。应先做：

* 计算 $$2 \times 2$$ 矩阵的特征值。
* 观察对角矩阵的特征值。
* 比较相似矩阵的特征多项式。
* 找一个不可对角化矩阵。
* 理解特征值、特征向量、特征子空间之间的区别。
* 再进入标准形和结构定理。

初学者的题目要密集、短小、反馈快。短题不是低级题，它们负责安装概念和工具的基本接口。

### 中级学习者应该选什么题？

中级学习者已经知道基本事实，但常常缺少工具选择能力。这个阶段的选题重点，应从“知道什么”转向“什么时候用什么”。

适合中级阶段的题，通常有几个特征：

* 题面不直接提示方法。
* 有两条以上可能路线。
* 需要比较条件强弱。
* 可以从特殊情形进入。
* 做完后能提炼出方法模板。
* 有值得保存的反例或边界。

例如分析学中的一个题目要求证明某个函数列一致收敛。若题面直接说“用 Weierstrass 判别法”，训练价值有限；若题目只给出函数列，需要学习者判断是用上确界估计、单调性、Dini 定理、Weierstrass 判别法，还是构造反例，训练价值更高。

中级阶段需要更多“方法触发题”。例如：

* 看到`紧致`，能否想到有限子覆盖或收敛子列？
* 看到`一致收敛`，能否想到上确界范数？
* 看到`可逆矩阵`，能否想到相似、消元、行列式非零？
* 看到`对称性`，能否减少变量？
* 看到`存在性`，能否想到构造、极值、紧致性、完备性或不动点？

这个阶段不应只追求题目数量。更重要的是做完题后能说出：这题真正训练了哪个工具？它和哪些旧题属于同一类？它的失败边界在哪里？

### 高级学习者应该选什么题？

高级训练的题目不再只是为了验证知识点，而是为了扩展问题空间。一个题目做完以后，应继续问：能否推广？能否弱化条件？能否找到反例？能否换一种语言？能否解释为什么这个方法到这里为止？

高级题通常有几个功能：

* 训练长证明组织。
* 训练方法边界意识。
* 训练跨领域连接。
* 训练提出新问题。
* 训练把直觉转化为可检查命题。
* 训练写作和交流。

例如一个线性代数命题已经证明了，可以继续追问：

* 这个结论在无限维空间中还成立吗？
* 若矩阵换成线性算子，需要哪些拓扑条件？
* 若域从 $$\mathbb{C}$$ 换成 $$\mathbb{R}$$，会发生什么？
* 若去掉可逆性，还能通过逼近处理吗？
* 这个结论是否其实是某个更一般结构的特例？

高级选题不一定更长，但要能生成后续问题。一个好题会像节点一样连接到多个方向，而不是做完就结束。

### 难度怎样控制？

题目难度不能只看“会不会做”。更准确的判断是：题目中未知部分占多少，能否通过有限提示进入，做完后是否能沉淀方法。

可以把题目分成四档：

| 难度   | 状态            | 用途           |
| ---- | ------------- | ------------ |
| 低难度  | 基本能独立完成       | 巩固事实、提高手感    |
| 合适难度 | 能开始，但会在关键处卡住  | 最适合训练        |
| 高难度  | 能理解题意，但无法形成路线 | 适合拆分、查提示、做专题 |
| 过高难度 | 题意、背景、工具都不清楚  | 暂时不适合作为主训练题  |

最有效的题，通常不是完全不会的题，而是“能进入，但做不完”的题。它能暴露具体卡点，又不至于让学习者只能被动接受答案。

判断题目是否合适，可以看三个信号：

* 能否写出题目对象、条件和目标？
* 能否做出至少一个特殊情形？
* 能否说出自己卡在哪里？

如果三个问题都答不上来，这道题暂时太高。应先补定义、例子、基础工具，或者把题目拆成小问题。

### 怎样从教材题、竞赛题和研究题中选？

不同来源的题目有不同功能。

教材题通常用于安装概念和标准工具。它们按章节排列，适合系统学习，但缺点是提示性太强：刚学完某个定理，后面的题很可能就是用这个定理。做教材题时，要主动去掉提示，问自己：如果不知道它在这一章，是否还能想到这个方法？

竞赛题通常用于训练技巧、构造、反例、变形和非标准入口。它们适合提高灵活性，但不宜替代系统学习。竞赛题的缺点是容易让学习者过度追求巧法，而忽视概念结构。

研究题或论文中的问题，功能不同。它们常常没有标准路线，甚至没有完整答案。做这类题时，目标不是快速解决，而是建立问题档案：背景是什么，已知结果是什么，简单情形怎样，失败路线有哪些，需要哪些工具，可能的推进方向是什么。

更稳妥的组合是：

* 教材题：保证系统性。
* 经典题：训练方法。
* 反例题：校准边界。
* 综合题：训练组织。
* 开放题：训练探索。

只做教材题，方法可能稳定但不灵活；只做竞赛题，技巧可能丰富但系统性不足；只看论文题，容易过早进入无反馈状态。题目来源要混合，但每类题的功能要清楚。

### 什么时候应该重做旧题？

重做旧题不是重复劳动。很多题第一次做时，学习者只是在追答案；第二次做时，才开始看结构；第三次做时，才能提炼方法。

值得重做的题通常有几类：

* 当时完全依赖答案的题。
* 用到重要工具的题。
* 有多种解法的题。
* 失败路线很多的题。
* 能代表一类问题的题。
* 后来学了新工具后，可以重新处理的题。

重做时不要背原答案。更有效的方式是只看题面，尝试复现入口。如果入口仍然想不起来，说明这道题还没有进入系统。此时应回到工具卡，而不是继续背答案。

旧题还适合做变式：

* 改参数。
* 改区间。
* 改维数。
* 改对象。
* 删掉一个条件。
* 加强一个结论。
* 换一种证明方法。

变式能检验方法是否真的掌握。原题会做，可能只是记忆；变式能做，才说明结构已经被吸收。

### 选题怎样避免碎片化？

碎片化选题的表现是：今天做一个极限，明天做一个矩阵，后天做一道组合题，每题都有收获，但彼此没有连接。短期看内容丰富，长期看难以形成工具库。

避免碎片化，需要围绕主题选题。一个主题可以是概念，也可以是方法。

按概念选题：

* `紧致性`专题。
* `一致收敛`专题。
* `特征值`专题。
* `同态`专题。
* `条件期望`专题。

按方法选题：

* 反例构造专题。
* 放缩估计专题。
* 极限交换专题。
* 可逆近似专题。
* 特殊情形到一般情形专题。
* 图像辅助猜想专题。

专题选题的好处是：题目之间会互相解释。第一题提供事实，第二题提供工具，第三题暴露反例，第四题展示边界，第五题连接到其他章节。这样做题不是横向铺开，而是向同一结构反复施压。

一个专题不必很长。5—10 道高质量题，配合复盘和工具卡，往往比 50 道随机题更有效。

### Part 3 的工具卡：选题决策表

| 判断项  | 问题          | 适合选择                  | 暂缓选择           |
| ---- | ----------- | --------------------- | -------------- |
| `目标` | 这道题训练什么？    | 能明确说出训练定义、工具、反例、综合或探索 | 只因为“看起来高级”     |
| `难度` | 能否进入题目？     | 能写出对象、条件、目标，并能做小版本    | 连题意和背景都不清楚     |
| `反馈` | 做完后能否知道哪里错？ | 有答案、提示、同伴、AI 或可计算检验   | 完全无反馈，且基础不足    |
| `连接` | 是否连接当前主题？   | 能接到正在学的概念或方法          | 与当前系统无关，只是随机遇到 |
| `产物` | 做完能留下什么？    | 可生成工具卡、反例卡、错因卡、题型模板   | 只能得到一个孤立答案     |
| `边界` | 是否能测试条件？    | 可以删改条件、构造反例、做变式       | 只能机械代入公式       |
| `迁移` | 是否代表一类问题？   | 做完能抽出结构               | 只能记住特殊技巧       |
| `阶段` | 是否符合当前水平？   | 略高于当前能力，需要努力但可诊断      | 远高于当前能力，只能看答案  |

### Part 3 的工具卡：一周选题配比

| 训练目标  |  题目数量 | 说明          |
| ----- | ----: | ----------- |
| 定义与例子 | 2—4 题 | 新概念必须先落地    |
| 基本计算  | 2—4 题 | 保持符号和技术熟练   |
| 工具触发  | 2—3 题 | 专门练一个方法何时出现 |
| 反例构造  | 1—2 题 | 防止概念边界模糊    |
| 综合题   | 1—2 题 | 训练路线组织      |
| 旧题复现  | 1—3 题 | 检查方法是否进入系统  |
| 变式题   | 1—2 题 | 检查是否能迁移     |

这个配比不需要机械执行。它的作用是防止训练失衡：只做基础题，工具增长慢；只做难题，反馈太少；只看答案，判断力不增长；只追新题，旧方法不稳定。合理选题应让事实、工具、直觉和复现同时发生。
